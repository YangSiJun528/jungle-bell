# Jungle Bell — 요구사항 정의서

크래프톤 정글 출석체크 리마인더

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | Jungle Bell |
| 프로젝트/폴더명 | jungle-bell |
| 목적 | 크래프톤 정글 출석체크 시간을 알려주고, 미처리 시 반복 경고하는 시스템 트레이 앱 |
| 플랫폼 | macOS, Windows |
| 기술 스택 | Tauri v2 (Rust 백엔드 + 경량 WebView 프론트엔드) |
| 외부 서버 | 없음. GitHub 릴리즈 확인(업데이트 체크)만 외부 통신 |
| 데이터 전송 | 없음. 모든 데이터는 로컬 저장 |

### 1.1 핵심 원칙: 자동 출석 금지

**이 앱은 자동 출석 기능을 제공하지 않으며, 앞으로도 제공하지 않는다.**

Jungle Bell은 출석체크를 "대신 해주는" 도구가 아니다. 출석체크는 반드시 사용자가 직접 수행해야 한다. 이 앱의 역할은 출석 시간을 놓치지 않도록 알림을 제공하고, 현재 출석 상태를 확인하여 보여주는 것에 한정된다.

구체적으로 다음 기능은 구현하지 않는다:

- 출석 버튼 자동 클릭 또는 자동 폼 제출
- 사용자 행위를 시뮬레이션하는 모든 기능
- 출석체크 페이지와의 쓰기(write) 상호작용

앱이 수행하는 서버 통신은 오직 출석 상태 확인(읽기 전용 GET 요청)과 토큰 갱신뿐이다.

## 2. 출석체크 시간대 정의

모든 시간은 KST(UTC+9) 기준이며, 내부 저장은 UTC로 한다.

### 2.1 시간대 구분

| 시간대 | KST 범위 | 의미 |
|--------|----------|------|
| 아침 출석 (학습 시작) | 04:00 ~ 10:00 | 이 시간 안에 학습 시작 체크를 해야 함 |
| 학습 중 | 10:00 ~ 23:00 | 정상적으로는 학습 진행 중인 시간 |
| 저녁 출석 (학습 종료) | 23:00 ~ 04:00 (익일) | 이 시간 안에 학습 종료 체크를 해야 함 |

### 2.2 사용자 커스텀

- 사용자가 각 시간대의 시작/종료 시각을 조정할 수 있어야 한다 (설정 화면).
- 사용자가 "이 시각까지 체크하겠다"는 자체 목표 시각을 설정할 수 있다. 이 시각을 초과하면 PENDING_URGENT로 전환된다.

## 3. 상태 머신

앱은 아침 출석과 저녁 출석 각각에 대해 독립적인 상태를 관리한다.

### 3.1 상태 정의

```
┌──────────────────┬──────────────────────────────────────────────────┐
│ 상태             │ 조건                                             │
├──────────────────┼──────────────────────────────────────────────────┤
│ IDLE             │ 현재 시간대에 해당하는 출석 이벤트가 없음         │
│ PENDING_RELAXED  │ 출석 가능 시간대이고, 여유 있음, 아직 미처리     │
│ PENDING_URGENT   │ 출석 가능 시간대이고, 마감 임박 또는              │
│                  │ 사용자 목표 시각 초과, 아직 미처리                │
│ DONE             │ 해당 출석 완료 확인됨                            │
│ OVERDUE          │ 출석 가능 시간대가 지났는데 미처리 (지각)         │
└──────────────────┴──────────────────────────────────────────────────┘
```

### 3.2 상태 전이

```
[시간대 진입] → PENDING_RELAXED
                    │
                    ├── 출석 확인됨 → DONE
                    │
                    ├── 마감 30분 전 또는 사용자 목표 시각 초과 → PENDING_URGENT
                    │       │
                    │       ├── 출석 확인됨 → DONE
                    │       │
                    │       └── 시간대 종료 → OVERDUE
                    │
                    └── 시간대 종료 → OVERDUE

OVERDUE ── 출석 확인됨 → DONE (지각이지만 완료)
```

### 3.3 URGENT 전환 기준

마감 임박의 기준은 다음 중 먼저 도달하는 조건:

- 시간대 종료까지 30분 이하 남음 (기본값, 사용자 조정 가능)
- 사용자가 설정한 목표 시각을 초과함

## 4. 출석 확인 메커니즘

### 4.1 웹 기반 확인

- 출석체크 서비스는 구글 계정 기반 로그인을 요구한다.
- 로그인 후 쿠키에 access token과 refresh token이 저장된다.
- 앱은 Tauri 내장 WebView를 백그라운드(숨김 상태)로 유지하며, WebView 내에서 주기적으로 출석 상태 페이지를 확인한다.

### 4.2 인증 흐름

WebView 자체 쿠키 저장소를 그대로 활용한다. 토큰을 별도로 추출하거나 Rust 백엔드에서 관리하지 않는다.

```
[최초 실행]
  사용자가 앱 내 WebView에서 구글 로그인
  → 출석체크 사이트 리다이렉트
  → 쿠키가 WebView 내부 저장소에 자동 저장

[이후 실행]
  WebView가 저장된 쿠키를 자동으로 사용
  → 주기적 페이지 요청 시 쿠키가 자동 첨부됨
  → 토큰 갱신도 사이트 측에서 쿠키 기반으로 처리

[세션 만료/로그아웃 감지]
  WebView에서 출석 페이지 요청 시 로그인 페이지로 리다이렉트 감지
  → "재로그인 필요" 알림 발송
  → 트레이 메뉴에서 재로그인 유도
```

### 4.3 WebView 운용

- WebView는 앱 실행 중 항상 백그라운드에서 숨김 상태로 유지한다.
- 화면에 표시하지 않되, 주기적으로 JavaScript를 실행하여 출석 상태 페이지를 fetch하고 결과를 파싱한다.
- WebView → Rust 백엔드로 상태 결과를 Tauri IPC(invoke/event)로 전달한다.
- 로그인이 필요한 경우에만 WebView를 사용자에게 표시한다.

### 4.4 확인 주기

| 상태 | 출석 상태 확인 주기 |
|------|---------------------|
| IDLE | 1시간마다 (시간대 진입 감지용) |
| PENDING_RELAXED | 10분마다 |
| PENDING_URGENT | 3분마다 |
| OVERDUE | 3분마다 |
| DONE | 1시간마다 (상태 변경 감지용) |

## 5. 알림 시스템

### 5.1 알림 채널

OS 네이티브 알림을 유일한 알림 채널로 사용한다. 별도 권한 요청은 알림 권한 하나뿐이다.

### 5.2 알림 빈도

아래는 기본값이며, 사용자가 설정에서 상태별 알림 주기를 자유롭게 조정할 수 있다.

| 상태 | 알림 주기 (기본값) | 알림 메시지 예시 |
|------|---------------------|------------------|
| PENDING_RELAXED | 30분마다 | "출석 가능합니다. 아침 출석을 진행해 주세요." |
| PENDING_URGENT | 5분마다 | "⚠️ 출석 마감이 임박합니다!" |
| OVERDUE | 3분마다 | "🚨 지각 중입니다! 지금이라도 출석해 주세요." |
| DONE | 알림 없음 | — |
| IDLE | 알림 없음 | — |

### 5.3 알림에 포함할 정보

- 현재 상태 (여유/임박/지각)
- 남은 시간 (해당 시간대 종료까지)
- 출석 페이지로 이동할 수 있는 동작 버튼 (가능한 경우)

### 5.4 방해 금지 모드 (선택 기능)

- 사용자가 특정 시간대에는 알림을 받지 않도록 설정할 수 있다 (예: 수면 시간).
- OS 수준의 방해 금지 모드가 활성화되면 앱도 이를 따른다.

## 6. 시스템 트레이

### 6.1 트레이 아이콘 상태

| 상태 | 아이콘 색상 | 설명 |
|------|-------------|------|
| IDLE | 회색 | 현재 출석 이벤트 없음 |
| PENDING_RELAXED | 초록 | 출석 가능, 여유 있음 |
| PENDING_URGENT | 노랑 | 출석 가능, 시간 임박 |
| OVERDUE | 빨강 | 지각 중 |
| DONE | 파랑 | 출석 완료 |

### 6.2 트레이 메뉴 항목

```
[아이콘] 출석체크 알리미
├── 상태: 아침 출석 대기 중 (09:23까지 37분 남음)
├── ─────────────────
├── 출석 페이지 열기        → 기본 브라우저에서 출석 페이지 오픈
├── 설정...                → 설정 윈도우 열기
├── ─────────────────
└── 종료
```

### 6.3 트레이 툴팁

마우스 호버 시: "아침 출석 대기 중 — 37분 남음"

### 6.4 macOS 특이사항

- 메뉴바에 항상 표시됨.
- 아이콘은 macOS 메뉴바 스타일에 맞게 템플릿 이미지(흑백)로 제공하고, 색상 표시는 메뉴 내 텍스트 또는 별도 표기로 대체하는 것을 고려한다.

### 6.5 Windows 특이사항

- 기본적으로 숨김 영역(^)에 배치됨.
- 첫 실행 시 "트레이 아이콘 항상 표시" 설정 가이드를 안내한다.
- 트레이가 숨겨져 있어도 OS 알림은 정상 동작한다.

## 7. 설정 화면

Tauri WebView로 구현하는 최소한의 설정 UI. 트레이 메뉴에서 "설정"을 클릭하면 열린다.

### 7.1 설정 항목

| 카테고리 | 항목 | 기본값 | 설명 |
|----------|------|--------|------|
| 계정 | 로그인 상태 | — | 현재 로그인 상태 표시, 로그인/로그아웃 버튼 |
| 시간 | 아침 출석 시작 | 04:00 KST | 커스텀 가능 |
| 시간 | 아침 출석 마감 | 10:00 KST | 커스텀 가능 |
| 시간 | 저녁 출석 시작 | 23:00 KST | 커스텀 가능 |
| 시간 | 저녁 출석 마감 | 04:00 KST | 커스텀 가능 |
| 시간 | 나의 목표 시각 (아침) | 미설정 | 이 시각 초과 시 URGENT 전환 |
| 시간 | 나의 목표 시각 (저녁) | 미설정 | 이 시각 초과 시 URGENT 전환 |
| 알림 | URGENT 전환 기준 | 마감 30분 전 | 커스텀 가능 |
| 알림 | RELAXED 알림 주기 | 30분 | 커스텀 가능 |
| 알림 | URGENT 알림 주기 | 5분 | 커스텀 가능 |
| 알림 | OVERDUE 알림 주기 | 3분 | 커스텀 가능 |
| 알림 | 방해 금지 시간대 | 미설정 | 선택 사항 |
| 시스템 | 시스템 시작 시 자동 실행 | OFF | OS 로그인 시 자동 시작 |
| 시스템 | 업데이트 확인 | ON | GitHub 릴리즈 확인 |

### 7.2 설정 저장

- 로컬 파일(JSON 또는 TOML)로 저장한다.
- 경로: OS별 표준 앱 데이터 디렉토리 (macOS: `~/Library/Application Support/jungle-bell/`, Windows: `%APPDATA%\jungle-bell\`)
- 인증 정보는 WebView 쿠키 저장소에서 관리하므로 설정 파일에 포함하지 않는다.

## 8. 프로젝트 구조

```
jungle-bell/
├── src-tauri/
│   ├── src/
│   │   ├── main.rs            # 앱 진입점, Tauri 빌더 설정
│   │   ├── state.rs           # 상태 머신 정의 및 전이 로직
│   │   ├── scheduler.rs       # tokio 기반 타이머/스케줄러
│   │   ├── checker.rs         # WebView에서 받은 출석 상태 처리
│   │   ├── notifier.rs        # OS 알림 발송
│   │   ├── tray.rs            # 트레이 아이콘/메뉴 관리
│   │   ├── config.rs          # 설정 파일 읽기/쓰기
│   │   └── updater.rs         # GitHub 릴리즈 업데이트 확인
│   ├── icons/                 # 상태별 트레이 아이콘 (5종)
│   ├── Cargo.toml
│   └── tauri.conf.json
├── src/                       # 프론트엔드 (설정 화면 + 출석 확인)
│   ├── index.html
│   ├── settings.js            # 설정 UI 로직
│   ├── checker.js             # 백그라운드 WebView에서 출석 상태 확인 및 IPC 전달
│   └── login.js               # 로그인 WebView 처리
├── package.json
└── README.md
```

## 9. 핵심 Rust 의존성

| 크레이트 | 용도 |
|----------|------|
| tauri (v2) | 앱 프레임워크, WebView 관리 |
| tauri-plugin-notification | OS 네이티브 알림 |
| tauri-plugin-autostart | 시스템 시작 시 자동 실행 |
| tokio | 비동기 런타임, 스케줄러 |
| serde / serde_json | 설정 파일 직렬화 |
| chrono / chrono-tz | 시간대 처리 (UTC ↔ KST) |

## 10. 빌드 및 배포

### 10.1 빌드

- Tauri v2 CLI로 macOS (.dmg) 및 Windows (.msi 또는 .exe) 빌드.
- CI: GitHub Actions에서 크로스 플랫폼 빌드.

### 10.2 업데이트

- GitHub Releases에 바이너리 게시.
- 앱 내에서 주기적으로 GitHub API를 호출하여 새 버전 확인.
- Tauri 내장 업데이터 플러그인(`tauri-plugin-updater`) 활용 가능.

### 10.3 서명

- macOS: Apple Developer 인증서 (없으면 Gatekeeper 경고 발생).
- Windows: 코드 서명 인증서 (없으면 SmartScreen 경고 발생).
- 인증서 없이 배포 시, 사용자에게 우회 방법 안내 필요.

## 11. 보안 고려사항

| 항목 | 대응 |
|------|------|
| 인증 정보 | WebView 내부 쿠키 저장소에서 관리 (별도 추출/저장 없음) |
| 네트워크 | 출석 사이트 및 GitHub API 외 통신 없음 |
| 데이터 수집 | 없음. 텔레메트리, 분석 데이터 일체 전송하지 않음 |
| WebView 범위 | 출석 사이트에만 접근. 백그라운드 WebView는 숨김 상태로 유지 |
